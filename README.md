  - cpu_usage_percent - загруженность

  - disk_usage_percent - заполненность диска в %

RAM
  - memory_usage_percent - как используется RAM
  - swap_usage_bytes - использовался ли SWAP

## Задание 5
PUSH плюсы:

  + Доступ агента к серверу: не нужен доступ сервера к агенту, что упрощает сетевую организацию и проще в аутентификации
  + Горизонтальное масштабирование
  + Может заранее считать результаты и готовить их

PUSH минусы:
  - Больше обслуживания агентов
  - Независимость агентов и отсутствие единой точки управления
  - Нужно фильтровать для избежания ДДОСа сервера с метриками
  - Надо проверять состояние каждого сервера, заходя на агента

PULL плюсы:

  + Единый центр настройки и просмотра состояния - сервер с доступом ко всем хостам
  + Автоматический discovery
  + Можно получить информацию "в моменте"
  + Можно работать без агентов
  + Удобная проверка на liveness (не ответил - упал)
  + Централизованная аутентификация

PULL минусы:
  - Надо правильно настраивать безопасность и доступ сервера к хостам
  - Сложнее горизонтальное масштабирование
  - Нагрузка на сеть, т.к. сервер ходит к хостам

Pull — для своих серверов, микросервисов, Kubernetes. Всё в одной сети, прямой доступ есть, сервисы долгоживущие.

Push — для чужого/распределённого: IoT, мобилки, serverless, клиентские устройства за NAT, batch-джобы. Сервер не может достучаться — агент шлёт сам.
